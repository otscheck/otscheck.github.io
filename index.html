Responsabilité Unique

Définition

Chaque module ou classe ne doit avoir qu'une seule raison de changer. C'est-à-dire qu'il ne doit avoir qu'une seule responsabilité bien définie.

Exemple sans le principe (non SOLID)

// Mauvais exemple
class Rapport {
  generatePDF() { /* générer le PDF */ }
  saveToFile() { /* enregistrer le fichier */ }
  sendByEmail() { /* envoyer le mail */ }
}

Exemple corrigé (SOLID)

// Bon exemple
class Rapport {
  generatePDF() { /* générer le PDF */ }
}

class FichierManager {
  saveToFile(rapport) { /* enregistrer */ }
}

class EmailService {
  send(rapport) { /* envoyer */ }
}
